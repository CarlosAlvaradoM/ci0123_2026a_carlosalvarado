# Plan de actualizacion

En este plan de actualizacion se pretender informar sobre como se va a trabajar en la simulacion del protocolo.
Que cosas o funciones se van a quedar, que se debe de eliminar o cosas nuevas.
Asi como un tiempo estimado de fechas / tiempo aproximadamente de cuando pueden estar listos.

## Cosas a eliminar

- Comandos CONNECT, DISCONNECT, PING, READ, LIST, SEARCH, OK, LISTDATA, NOTFOUND, ERROR

Aproximadamente 9 dias

## Cosas a mantener

- El sistema de comandos con parametros
- El comando GET

Aproximadamente en 10 dias

## Cosas a incorporar

- Comandos con codigos
- Comandos POST, PATCH, DELETE
- Posibles errores
- Broadcats a server para tabla

Aproximadamente 11 dias
